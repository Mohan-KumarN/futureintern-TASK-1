# Sales Performance Analytics (Python)

A small Python data-analysis pipeline that generates synthetic sales data, analyzes it, and creates charts in the `assets/` folder.

## Project Structure

- `generate_data.py` — generates `sales_data.csv` (synthetic dataset)
- `analyze_sales.py` — reads `sales_data.csv`, prints summary stats, and generates charts
- `sales_data.csv` — input dataset (generated by `generate_data.py`)
- `assets/` — output images
  - `revenue_trends.png`
  - `top_products.png`
  - `category_revenue.png`
  - `regional_performance.png`
  - `category_profit.png`
- `Sales_Performance_Report.md` — the final written report (charts referenced from `assets/`)
- `task details.txt` — task description/instructions

## Requirements

- Python 3.x

Install dependencies:

```bash
pip install pandas numpy matplotlib seaborn
```

## Usage

### 1) (Optional) Generate/Regenerate the dataset

```bash
python generate_data.py
```

Creates (or overwrites):

- `sales_data.csv`

### 2) Run the analysis + chart generation

```bash
python analyze_sales.py
```

It will:

- read `sales_data.csv`
- print a missing-value check + summary statistics
- generate charts in `assets/`:
  - `assets/revenue_trends.png`
  - `assets/top_products.png`
  - `assets/category_revenue.png`
  - `assets/regional_performance.png`
  - `assets/category_profit.png`

## Outputs

### Charts

Stored in:

- `assets/`

### Written Report

Stored in:

- `Sales_Performance_Report.md`

Note: the scripts generate images and console summary output; the Markdown report is already present in this repository and references the generated assets.

## Troubleshooting

- If `analyze_sales.py` fails due to missing input, ensure `sales_data.csv` exists (run `generate_data.py`).
- If plotting fails, confirm required packages are installed (`pandas`, `numpy`, `matplotlib`, `seaborn`).
